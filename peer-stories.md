# Library A — Peer Stories

Named-company AI transformation stories. Qualitative evidence for the FlightPlan transformation narrative. See `README.md` for disciplines (credibility tiering, currency, no-fabrication, de-duplication).

## Tags

Each entry is tagged so a FlightPlan can pull the relevant subset:

- **Vertical / operating model** — e.g. `precision-contract-manufacturing`, `med-surg-hospital`, or `cross-vertical` for general leadership stories.
- **Function** — the operational area the story is about (procurement, quality, customer service, enterprise-wide, etc.).
- **Company size** — small / mid-market / enterprise.
- **AI work type** — agentic automation, enterprise search, document AI, machine-data analytics, operating-model redesign, etc.
- **Story type** — process win / operating-model shift / new-revenue / scaling / leadership.
